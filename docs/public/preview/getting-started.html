Primeiros passos com o Guardia DS

Adicione o @guardiafinance/design-system em qualquer app React 19 em quatro passos. Traz 45 componentes acessíveis, tokens semânticos alinhados com a marca, Tailwind 4 CSS-first e Poppins carregada automaticamente. Lastica fica fora do bundle — é reservada a logos.

Instalação

~5 min
1

Configure o registry do GitHub Packages

O pacote é privado. Gere um Personal Access Token com scope read:packages e crie um .npmrc na raiz do seu projeto:

# .npmrc
@guardiafinance:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=${GITHUB_TOKEN}

Em CI, exporte GITHUB_TOKEN com um token de read-only nos secrets.

2

Instale o pacote e as peer deps

# o pacote
npm install @guardiafinance/design-system

# peer deps
npm install react@^19 react-dom@^19 react-router@^7 zod@^4

Se a resolução reclamar de peer-deps, use --legacy-peer-deps.

3

Importe o CSS no entrypoint da aplicação

// src/main.tsx (ou equivalente)
import "@guardiafinance/design-system/styles.css";

Esse único import já traz todos os tokens de cor, tipografia, spacing, radii, shadows e motion, além do reset e da CDN do Google Fonts (Poppins + Roboto).

4

Envolva a app no ThemeProvider

import { ThemeProvider, ThemeToggle } from "@guardiafinance/design-system";

export function App() {
  return (
    <ThemeProvider defaultTheme="light">
      <Shell>...</Shell>
      <ThemeToggle/>
    </ThemeProvider>
  );
}

Tema é controlado via atributo data-theme="light|dark" no <html>. Pronto.

Seu primeiro componente

exemplo mínimo 
import { Button, Card, Badge } from "@guardiafinance/design-system";
import { Save, Sparkles } from "lucide-react";

export default function Dashboard() {
  return (
    <Card>
      <Card.Header>
        <Card.Title>Saldo total</Card.Title>
        <Badge variant="secondary">Real-time</Badge>
      </Card.Header>
      <Card.Content>
        <Button variant="default" leadingIcon={<Save/>}>Salvar</Button>
        <Button variant="secondary" leadingIcon={<Sparkles/>}>Perguntar ao Isac</Button>
      </Card.Content>
    </Card>
  );
}

variant="default" é a ação primária (laranja Guardia) · variant="secondary" é a ação de autoridade (violeta) · ícones vêm do lucide-react.

Tailwind 4 no seu app

opcional, mas recomendado
Se o seu app não usa Tailwind, você ainda recebe todos os estilos dos componentes (o CSS vem compilado no styles.css). Se usa Tailwind 4, os tokens Guardia ficam disponíveis como utilities — bg-primary, text-foreground, bg-guardia-orange-500, shadow-brand, duration-base ease-standard — sem configuração adicional.
